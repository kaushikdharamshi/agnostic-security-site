The Problem

AI Agents Have Unrestricted Access

Copilot, Claude Code, and Cursor run on developer machines with full access to .env files, API keys, SSH keys, customer PII, and source code. No guardrails.

🔓

Credential Exposure

AI assistants can read .env, .pem, id_rsa, and credentials files — then include them in completions or send them to cloud APIs.

📤

Data Exfiltration

Shell commands like curl -d @secrets.json, printenv, or cat ~/.ssh/id_rsa can leak data before anyone notices.

⚖️

Compliance Risk

PCI-DSS, SOX, HIPAA — all require strict controls over sensitive data access. AI agents bypass every one of them.

Architecture

Seven Layers of Defense in Depth

Every AI interaction passes through multiple independent security layers. Any single layer can fail and others still protect.

Features

What AgnosticSecurity Protects

📁

File Access Gate

2-layer path + content DLP. Blocks .env, .pem, credentials, SSH keys, and any file containing PII — before the AI sees it.

⌨️

Exec Command Guard

20+ regex patterns with LLM verification. Blocks cat ~/.env, printenv, curl exfiltration, and obfuscated commands.

🧠

Prompt Intent Analysis

4-layer detection: PII regex → intent keywords → Pydantic rules → local Ollama LLM. Catches "get me all SSNs" before it executes.

🔐

PII Redaction

Detects and redacts email, phone, SSN, credit cards, API keys in both inputs and outputs. Modifies in-place — doesn't block.

🖥️

System Prompt Masking

Strips hostname, OS, username, and repo paths from system prompts before they reach the LLM. No infrastructure leakage.

📸

Media Filtering

Blocks image, audio, and video attachments from multi-part messages. Prevents visual PII leakage via screenshots.

🔑

Session Encryption

AES-256-GCM encryption for audit log entries and session data at rest. PBKDF2 key derivation.

📊

Breach Compliance

13 fintech breach types (PCI-DSS, SOX, HIPAA). Immutable SHA-256 audit log. Real-time dashboard with webhook alerting.

🔌

Auto-Instrumentation

Zero-code monkey-patching for OpenAI, Anthropic, and LangChain SDKs. Every LLM call monitored automatically.
